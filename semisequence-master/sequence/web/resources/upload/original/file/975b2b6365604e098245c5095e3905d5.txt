4월 11일부터 4월 29일까지 팀원들과 진행한 프로젝트를 종료한 시점에서 현재의 감정은 만족보다는 아쉬움이 많이 남는 것 같다.

프로젝트를 처음 시작할 때만 해도 한번도 접해보지 못한 부분이기에 감 조차 잡지 못했다.
수업시간에 처음 배워본 언어와 프로그램 등을 사용하여 어떤 프로젝트를 진행할 수 있는지, 개발 할 수 있는 기능은 어떤 것이 있는지 몰라서 막막했었는데
약 3주간의 프로젝트를 진행하면서 나보다 잘하는 팀원들의 도움을 받아 주제선정부터 모델링, 기능 개발, 프로젝트 발표까지 어떤식으로 진행되는지 전체적인 흐름을 잡게 된 것 이 가장 좋았던 것 같다.

프로젝트 진행간 스스로의 목표는
살면서 처음으로 해보는 프로젝트이고 시간이 지나서 보면 정말 기초적인 단계라고 생각을 했고,
모든일을 진행할 때 기초가 탄탄해야 무너지지 않는다는 생각이 강해서 나 스스로도 이번 프로젝트의 목표를 정확한 이해에 중점을 두고 진행한 것 같다.
모델링 단계를 진행하면서 내가 왜 유스케이스 다이어그램을 작성하는지, 왜 상관모델링을 작성하는지 등 내가 진행한 행위 하나하나를 왜하고 언제하는 단계인지 원초적인 이해를 하면서 작성했고,
기능 개발시에도 동일하게 코드를 이해하고 더 좋은 방법으로 작성할 수 없는지 고민했던 것 같다.

팀원으로서의 목표는 
원활한 소통과 열심히 하는 것이 중요하다고 생각했던 것 같다.
내가 활발하게 뭐든 잘 참여하는 성격이 아니고 시간을 가지고 신중하게 생각을 정리한 후 말하는 성격이기도 하고, 잘 이해하지 못한 부분에서 의견을 내면 안좋지 않을까라는 생각에 조심스러웠는데
시간이 지나면서 프로젝트의 전반적인 이해가 생겼고고 의사소통의 중요함을 더 느껴서 많이 참여하려고 노력했던 것 같다.

아쉬움이 남는점은 앞으로도 어떤 프로젝트를 진행할 때 끝나면 당연히 뿌듯하고 만족하겠지만,
진행한 프로젝트에 있어 끝나고 나면 더 다양하고 섬세한 결과물을 만들 수 있을 것 같다는 생각에 아쉬움이 남는 것 같고
